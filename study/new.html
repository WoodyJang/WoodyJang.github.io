    # 深入理解new操作符

    new 运算符在MDN上的描述为：

    new 运算符创建一个用户定义的对象类型的实例或具有构造函数的内置对象的实例。new 关键字会进行如下的操作：

    1. 创建一个空的简单JavaScript对象（即{}）；
    2. 链接该对象（即设置该对象的构造函数）到另一个对象 ；
    3. 将步骤1新创建的对象作为this的上下文 ；
    4. 如果该函数没有返回对象，则返回this

    # 剖析 new 的内部原理

    下面来举例深入讲解，来个例子

    function Parent(name){
    this.name = name
    say = () => {
        console.log('111')
    }
}

Parent.color = "black";

Parent.prototype.say = function(){
    console.log("I'm " + this.name);
 };
let child = new Parent("hahaha");

console.log(child.name) //hahaha
console.log(child.color) //undefined
child.say(); //I'm hahaha


console.log(Parent.name) //hahaha
console.log(Parent.color) //black
Parent.say(); //Parent.say is not a function

    JS引擎在执行这段代码时其实做了好多工作，伪代码如下：

    new Parent('hahaha') = {
    var obj = {};
    obj.__proto__ = Parent.prototype;
    var result = Parent.call(obj,"hahaha");
    return typeof result === 'object'? result : obj;
}

    将上述流程分为4个步骤来理解：

    1. 创建一个空对象obj

    2. 把 obj 的__proto__ 指向构造函数 Parent 的原型对象 prototype，此时便建立了 obj 对象的原型链：++obj->Parent.prototype->Object.prototype->null++。

    3. 在 obj 对象的执行环境调用 Parent 函数并传递参数"hahaha" 。 相当于 ++let result = obj.Parent("hahaha")++。 当这句执行完之后，obj 便产生了属性 name 并赋值为 "hahaha"。

    4. 考察第 3 步的返回值，如果无返回值 或者 返回一个非对象值，则将 obj 作为新对象返回；否则会将 result 作为新对象返回

    根据以上过程，我们发现 child 其实就是步骤4的返回值，

    • child 的原型链是：++child->Parent.prototype->Object.prototype->null++
    • child 上新增了一个属性name
    • child.name - 在步骤3中，obj 对象就产生了 name 属性。因此 child.name 就是这里的 obj.name
    • child.color - child 对象先查找自身的 color，没有找到便会沿着原型链查找，在上述例子中，我们仅在 Parent 对象上定义了 color，并没有在其原型链上定义，因此找不到。
    • child.say - child会先查找自身的 say 方法，没有找到便会沿着原型链查找，在上述例子中，我们在 Parent 的 prototype 上定义了say，因此在原型链上找到了say 方法。

    对于Parent来说，它本身也是一个对象，因此它在访问属性和方法时也遵守上述查找规则，所以：

    • Parent.color > "black"
    • Parent.name > "Parent"
    • Parent.say() > Parent.say is not a function

    Parent 的原型链： ++Parent->Function.prototype->Object.prototype->null++

    由于 Parent 的原型链上也没有定义 say 方法，因此返回异常提示。

    # 探索 new 的意义

    对 new 运算符有了较深入的理解之后，我们再回到开篇提到的问题：在JavaScript 中，万物皆对象，为什么还要通过 new 来产生对象？

    要弄明白这个问题，我们首先要搞清楚 child 和 Parent 的关系：

    • child 继承了 Parent 对象

    通过上面的分析我们发现， cat 通过原型链继承了 Animal 中的部分属性，因此我们可以简单的认为：Animal 和 cat 是继承关系。

    • child 是 Parent 的实例

    child 是通过 new 产生的对象，那么 child 到底是不是 Parent 的实例对象？

    我们先来了解一下JS是如何来定义实例对象：

    A instanceof B

    如果上述结果为true，JavaScript 认为 A 是 B 的实例对象，我们用这个方法来判断一下 child 和 Parent

    child instanceof Parent; //true

    从结果看，child 确实是 Parent 实例，要想更加证实这个结果，我们再来了解一下 instanceof 的内部原理：

    var L = A.__proto__;
var R = B.prototype;
if(L === R)
    return true;

    如果 A 的__proto__ 等价于 B 的 prototype，就返回 true 。 在 new 的执行过程2 中，child 的 proto 指向了Parent 的 prototype，所以 child 和 Parent 符合 instanceof 的判断结果。

    因此，通过 new 创建的 对象 和 构造函数 之间建立了一条原型链，原型链的建立，让原本孤立的对象有了依赖关系和继承能力，让JavaScript 对象能以更合适的方式来映射真实世界里的对象，这是面向对象的本质。

    # 实战演练

    function Foo() {
  getName = function () {
    console.log(1)
  }
  return this;
}
Foo.getName = function () {
  console.log(2)
}
Foo.prototype.getName = function () {
  console.log(3)
}
var getName = function () {
  console.log(4)
}
//声明式函数，在代码运行前会被提升到最顶部；
function getName() {
  console.log(5)
}

// ouput:
Foo.getName();           //2
getName();               //4
Foo().getName();         //1
getName();               //1
new Foo.getName();       //2
new Foo().getName();     //3
new new Foo().getName(); //3

    # new 操作符 实现

    const isComplexDataType = obj => (typeof obj === 'object' || typeof obj === 'function') && obj !== null
const selfNew = function(fn,...rest){
  let instance = Object.create(fn.prototype)
  let res = fn.apply(instance,rest)
  return isComplexDataType(res) ? res : instance
}

    /**
 * 递归遍历left的原型链与right的原型进行比较
 * @param {*} left 
 * @param {*} right 
 */
const selfInstanceof = function(left ,right){
  let proto = Object.getPrototypeOf(left)
  while(true){
    if(proto === null) return false
    if(proto === right.prototype) return true
    proto = Object.getPrototypeOf(proto)
  }
}
